現状メモ（適宜状態の追加を行うこと）
====================

WordPress プラグインを ZIP 化して導入後、以下の現象が発生。

- 固定ページを直接表示すると /#/ にアクセスされ、ヘッダーとサイドバーのみ表示される。
- サイドバーから「CSVインポート」「CSVエクスポート」を選択すると画面は正しく表示される。
- 「データベース一覧」を選択すると /#/db に遷移し、一瞬ページが表示されるがすぐに空白になる。
- 「操作ログ」を選択すると /#/logs に遷移するが、何も表示されない。
- 画面が空白になるとリロードが効かないため、ブラウザのアドレスバーから直接 / に移動し直す必要がある。

npm run build 実行後に生成ファイルを反映しても状況は変わらなかった。

今後同じ手順を繰り返さないよう、この状態を記録しておく。
